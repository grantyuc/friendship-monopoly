# 友誼大富翁 — 開發需求歷史記錄

本文件記錄所有開發過程中的使用者需求與設計決策，供未來開發者或 AI agent 參考。

---

## 初始需求（教案設計）

**專案背景**：製作單機網頁互動遊戲「友誼大富翁」，供國小4–6年級班級情感教育課程使用。

**教案目標**：
- 透過任務設計讓參與者反思生活中會遇到的情緒
- 幫助參與者理解同學夥伴的心情
- 依據4–6年級學員特別制定，意在增進彼此夥伴/同學間的愛心、互動及情誼

**遊戲進行方式**：兩隊以擲骰子方式在大富翁地圖上行走，完成地圖上的任務（分紅綠黃與機會命運），佔領土地，對手經過被佔領的土地時賺取積分，最後結算積分高之組別獲勝。

**棋盤格子設計**：
- 黃色（通力合作）11格：合作互動挑戰
- 紅色（情境表演，原名「挑戰情境」）10格：生活情境演戲
- 綠色（心情分享，原名「心靈補給」）11格：情緒分享與安慰練習
- 機會 4格 + 命運 4格（共8個隨機事件格）

**積分規則**：
- 每隊初始 1500 分
- 完成格子/經過起點 +100
- 走到對方已完成的格子 -100
- 重複踩到己方完成格子可再次挑戰，成功蓋小房子
- 走到對方有房子的格子，一個房子多減 100（2棟 -300）

## 技術需求（按時間順序）

### 1. 初始建置
- 製作可在本地開啟的單機網頁互動遊戲
- 使用 React + TypeScript + Vite + Tailwind
- 輸出單一 HTML 檔案，無需伺服器

### 2. 修正 file:// 問題
- 錯誤：`Uncaught SyntaxError: Unexpected identifier 'object'`
- 原因：Vite 預設輸出 ES module（`type="module"`），在 `file://` 協議下有限制
- 解決：改用 IIFE 格式（`format: 'iife'`、`target: 'es2015'`）

### 3. UI 修正（三項合一）
- 首頁命運格圖例要是黑色方塊（原本是白色）
- 首頁要提示可以輸入隊名（✏️ 點擊下方輸入框可修改隊名）
- 「再玩一次」要記住上次輸入的隊名

### 4. 按鈕等寬
- 首頁「開始遊戲」與「抽牌模式」按鈕要一樣大
- 解法：兩個按鈕都用 `width: clamp(200px, 28vw, 360px)`

### 5. 格子名稱更改
- 「挑戰情境」→「情境表演」
- 「心靈補給」→「心情分享」
- 需同步修改：BOARD_CELLS、CardModal type label、questions.ts 的題目卡標籤

### 6. 專案整理（第一次）
- 儲存所有必要資源在 session scratchpad 以便建置
- 清理用過的暫存檔
- 整理專案資料夾
- 重新建置驗證
- 輸出 README.md 供未來開發者參考

### 7. 動畫功能（兩項）
- 機會或命運效果的擲骰子也要有動畫（後退骰子）
- 移動也要有動畫（effect_moving phase + displayPositions + animCancelRef）

### 8. 效果落地觸發（兩項，開發中途補充）
- 機會或命運效果移動到的格子，也要觸發那一格的效果（自己的可以挑戰蓋房子，對方的要付錢）
- 踩到對方領地減少的分數要加在對方分數上面

### 9. MOVE_COMPLETE 分數轉移
- 正常骰子落地踩到對方領地，扣除分數也要加到對手（與 EFFECT_MOVE_COMPLETE 行為一致）

### 10. 摧毀建築音效
- 摧毀對手建築時要有對應音效

### 11. 後退骰子動畫統一
- 機會命運效果導致的擲骰子動畫要與普通回合一致（3D 動畫）
- 解法：提取 DiceFace 為共用元件，EffectModal 使用相同的 requestAnimationFrame 旋轉動畫

### 12. 機會命運效果修正（前進/後退/回到原點沒觸發）
- 問題根本原因：composite 效果迴圈中，move_forward 設完 phase:'effect_moving' 後，下一個 score_change 會覆蓋回 'rolling'
- 解法：composite 中把移動效果（move_forward/backward/to_start）分類為「延遲執行」，等其他即時效果跑完再最後套用
- 同時修正 extra_turn：需清除 diceValue 否則 showNextTurn 為 true 顯示換隊按鈕而非擲骰按鈕

### 13. 暫停回合觸發格子效果
- 被暫停的那一回合也要觸發格子的事件
- 解法：在 GameState 加 skipTurnReturn 欄位；NEXT_TURN 偵測到暫停時切換到暫停隊並執行完整落地邏輯；gameReducer 外層 wrapper 當 phase 回到 rolling 且 skipTurnReturn 有值時自動切換回待機隊

### 14. 專案整理（第二次）＋需求記錄
- 儲存所有必要資源、清理暫存檔、重新建置
- 更新 README.md 反映最新狀態
- 將本文件（PROMPTS.md）建立為開發需求歷史記錄

## 已知的設計決策

| 決策 | 原因 |
|------|------|
| IIFE 格式輸出 | 解決 file:// 協議下 ES module 的限制 |
| 建置在 scratchpad 執行 | macOS 掛載的 workspace 資料夾在 dist/ 清理時有 EPERM 錯誤 |
| composite 移動效果延遲執行 | 避免 score_change 覆蓋 effect_moving phase |
| skipTurnReturn 外層 wrapper | 暫停回合可能觸發多種後續效果（卡片、移動等），統一在 rolling 時自動換隊最簡潔 |
| extra_turn 清除 diceValue | showNextTurn 依賴 diceValue !== null，不清除會顯示換隊按鈕 |
| 分數轉移（deducted 計算） | 確保玩家分數不會低於 0，同時對手獲得的是實際被扣除的金額 |
